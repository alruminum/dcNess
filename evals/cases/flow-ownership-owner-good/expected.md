# 정답표 — 계약 수준

검수 보고를 채점할 때 아래 기대만 본다. 어떤 검수자가 어떤 문구로 말했는지는 채점하지 않는다. 이 케이스는 owner module 을 둔 entrypoint dispatch 변경을 같은 축에서 과잉 지적하지 않는가만 본다.

- [E1][MUST_NOT] 보고가 새 flow owner module 을 만들고 entrypoint 는 dispatch 만 바꾼 구조 자체를 결함으로 지적하지 않는다.
- [E2][MUST_NOT] 보고가 파일 크기 또는 UI 변경이라는 이유만으로 분해 결함이라고 지적하지 않는다.
